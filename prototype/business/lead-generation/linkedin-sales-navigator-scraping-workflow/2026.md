# LinkedIn Sales Navigator Scraping Workflow

## Purpose

This recipe produces a cleaned, deduplicated CSV of 200-2,000 leads extracted from LinkedIn Sales Navigator, matching your ICP filters — with names, titles, companies, LinkedIn URLs, and optionally work emails. The output is ready for enrichment pipeline ingestion or direct CRM import.

## Prerequisites

- [ ] **ICP definition** available from `persona-builder` agent — [ICP Definition](/business/customer-research/ideal-customer-profile-definition/2026)
- [ ] **LinkedIn Sales Navigator** — active Core ($99/mo) or Advanced ($149/mo) subscription
- [ ] **PhantomBuster account** — API key from [phantombuster.com](https://phantombuster.com) (free tier: 2hr/day)
- [ ] **Sales Navigator cookie** — extracted from browser session (required for PhantomBuster)

## Constraints

- LinkedIn allows ~80-150 profile views/day on Sales Navigator accounts. Exceeding this triggers temporary restrictions. [src1]
- Connection requests capped at ~100/week across all LinkedIn account types. [src1]
- PhantomBuster processes ~1 profile every 30 seconds. A batch of 500 profiles takes ~4 hours. [src2]
- GDPR applies to personal data of EU residents regardless of where your company is located — B2B work emails are generally permissible under legitimate interest. [src1]
- Apify community scrapers are limited to ~500 profiles per session before LinkedIn detection kicks in. [src2]

## Execution Flow

### Step 1: Build Sales Navigator Boolean Search

**Duration**: 10-15 minutes
**Tool**: LinkedIn Sales Navigator

Translate your ICP into Sales Navigator boolean search queries. Use Lead filters (not Account filters) for individual contacts.

```
Example boolean search for SaaS VP Sales at mid-market companies:

Title: (VP OR "Vice President" OR Director OR Head) AND (Sales OR Revenue OR "Business Development")
Company headcount: 51-500
Industry: Computer Software, Internet, SaaS
Geography: United States
Seniority: VP, Director
```

Save the search in Sales Navigator for future re-runs. Copy the full search URL — this is your input for the scraper.

**Verify**: Search returns 200-5,000 results. If fewer than 200, broaden criteria. If more than 5,000, narrow with additional filters.
**If failed**: Add or remove filters incrementally. Check that boolean operators are uppercase (AND, OR, NOT).

### Step 2: Extract Sales Navigator Cookie

**Duration**: 2-3 minutes
**Tool**: Browser DevTools

PhantomBuster and Apify require your Sales Navigator session cookie to operate.

```
Steps:
1. Open Sales Navigator in Chrome
2. Press F12 → Application tab → Cookies → linkedin.com
3. Find the cookie named "li_at"
4. Copy its value (long alphanumeric string)
5. Paste into PhantomBuster's LinkedIn Session Cookie field
```

**Verify**: Cookie value starts with "AQ" and is 200+ characters long.
**If failed**: Log out and back into LinkedIn, then re-extract. Cookies expire every 30-90 days.

### Step 3: Configure and Run Scraper

**Duration**: 5-10 minutes setup + 30-120 minutes runtime
**Tool**: PhantomBuster (Sales Navigator Search Export phantom)
**Rate limit**: Max 80-150 profiles/day per LinkedIn account. Spread across multiple sessions if needed. [src1]

```
PhantomBuster Configuration:
- Phantom: "Sales Navigator Search Export"
- Session cookie: [paste li_at value]
- Search URL: [paste saved search URL from Step 1]
- Number of profiles: 500 (recommended per run)
- Behavior:
  - Time between launches: 60 minutes minimum
  - Lines per launch: 100
  - Add email discovery: Yes (uses email credits)
```

**Expected output**: CSV with columns — linkedInProfileUrl, firstName, lastName, title, companyName, location, connectionDegree, sharedConnections
**Verify**: Row count matches expected range. No columns with 100% null values.
**If failed**: If account restricted, pause 24 hours. Reduce lines per launch to 50. Rotate to backup LinkedIn account if available.

### Step 4: Clean and Deduplicate

**Duration**: 10-15 minutes
**Tool**: Google Sheets, Excel, or Python

```python
import pandas as pd

df = pd.read_csv('phantombuster_export.csv')

# Remove duplicates by LinkedIn URL
df = df.drop_duplicates(subset=['linkedInProfileUrl'])

# Remove rows missing critical fields
df = df.dropna(subset=['firstName', 'lastName', 'companyName'])

# Normalize titles for consistency
df['title'] = df['title'].str.strip().str.title()

# Filter to ICP-matching titles only
icp_titles = ['vp', 'vice president', 'director', 'head']
df = df[df['title'].str.lower().str.contains('|'.join(icp_titles))]

df.to_csv('cleaned_leads.csv', index=False)
print(f"Cleaned: {len(df)} leads from {len(pd.read_csv('phantombuster_export.csv'))} raw")
```

**Verify**: Dedup reduces list by less than 10%. If more, search overlap between runs is too high.
**If failed**: Check that the linkedInProfileUrl column is present and correctly formatted.

### Step 5: Export Final Lead List

**Duration**: 5 minutes

Generate the final deliverable in CRM-ready format.

```python
output = df[['firstName', 'lastName', 'title', 'companyName',
             'linkedInProfileUrl', 'location', 'email']].copy()
output.columns = ['First Name', 'Last Name', 'Job Title', 'Company',
                   'LinkedIn URL', 'Location', 'Email']
output['Lead Source'] = 'LinkedIn Sales Navigator'
output['Date Scraped'] = '2026-03-11'
output.to_csv('final_leads.csv', index=False)
```

**Output files**:
- `final_leads.csv` — Primary lead list, sorted by company name
- `search_queries.txt` — Saved boolean searches for reproducibility

## Quality Benchmarks

| Quality Metric | Minimum Acceptable | Good | Excellent |
|---------------|-------------------|------|-----------|
| ICP title match rate | > 70% | > 85% | > 95% |
| Data completeness (name + company + title) | > 90% | > 95% | > 99% |
| Email discovery rate | > 30% | > 50% | > 70% |
| Duplicate rate post-dedup | < 10% | < 5% | < 2% |

**If below minimum**: Refine Sales Navigator boolean search. Overly broad searches dilute ICP match rate.

## Error Handling

| Error | Likely Cause | Recovery Action |
|-------|-------------|----------------|
| LinkedIn account restricted | Exceeded daily profile view limit | Pause 24 hours, reduce batch to 50 profiles/launch |
| PhantomBuster "cookie expired" | Session cookie invalidated | Re-extract li_at cookie from browser |
| Empty CSV output | Search URL malformed or cookie rejected | Verify search URL loads in browser, re-paste cookie |
| Partial extraction (stopped mid-run) | PhantomBuster runtime limit hit | Upgrade plan or split into multiple smaller runs |
| Email discovery returns 0% | LinkedIn privacy settings blocking | Use enrichment pipeline as fallback for email discovery |

## Anti-Patterns

### Wrong: Running scraper at maximum speed without delays
Blasting 1,000 profile views in an hour triggers LinkedIn's automated detection. Account gets restricted for 7-30 days, losing access to all saved leads and InMails. [src1]

### Correct: Throttle to 50-100 profiles per session with 60+ minute gaps
Spread extraction across multiple sessions. PhantomBuster's built-in rate limiting helps, but set conservative launch intervals. [src1]

### Wrong: Using personal LinkedIn account for scraping
If the account gets restricted, you lose your professional network and messaging history. Recovery is uncertain.

### Correct: Use a dedicated Sales Navigator seat for scraping
Separate scraping activity from your primary professional account. If restricted, only the scraping seat is affected. [src4]
