# Outreach Sequence Loading

## Purpose

This recipe configures and launches cold outreach sequences in Instantly, Lemlist, Apollo, or HubSpot Sequences with A/B test variants, sending schedules, and reply tracking. The output is a live campaign with leads loaded, 2-4 email variants per step configured, warmup validated, and performance monitoring active — ready to generate replies and booked meetings.

## Prerequisites

- [ ] **CRM-ready lead list** — formatted CSV from [Lead List Output Formatting](/business/lead-generation/lead-list-output-formatting/2026)
- [ ] **Email templates** — 2-4 variants per step with personalization variables (`{{firstName}}`, `{{companyName}}`, `{{personalization_line}}`)
- [ ] **Warmed sending accounts** — Google Workspace or Outlook accounts with 14+ days warmup completed
- [ ] **Sending domain** — separate from primary business domain (e.g., `outreach-{company}.com`) with SPF, DKIM, DMARC configured
- [ ] **Outreach tool account** — Instantly, Lemlist, Apollo, or HubSpot with active subscription
- [ ] **Compliance ready** — physical address for CAN-SPAM, unsubscribe mechanism, suppression list loaded

## Constraints

- New sending domains must warm up for 14-21 days before any cold outreach. Start at 20-30 emails/day and ramp by 10-20% weekly. Skipping warmup causes immediate spam folder placement. [src5]
- A/B tests require minimum 100-150 recipients per variant for statistical significance. Apollo's internal data shows meaningful patterns emerge around 200 recipients total (100 per variant). [src4]
- CAN-SPAM requires a physical mailing address and one-click unsubscribe in every cold email. GDPR requires legitimate interest basis for B2B outreach to EU contacts. [src7]
- Instantly campaign sequences use a single sequence array — only the first element is read. Add all steps within that single sequence. [src1]
- Lemlist campaign steps cannot be rearranged after launch. Build the complete sequence in draft before activating. [src2]
- Apollo recommends minimum 5 touchpoints per sequence. Adding one diversified touchpoint beyond auto-email increases meeting booking rate by up to 14%. [src3]

## Execution Flow

### Step 1: Connect and Verify Sending Accounts

**Duration**: 10 minutes
**Tool**: Outreach platform settings

Connect warmed email accounts to the outreach tool. Each tool requires different setup:

**Instantly**: Settings > Email Accounts > Connect > Google/Outlook OAuth. Enable Instantly's warmup network. Set daily sending limit per account (start 30-50/day). Verify deliverability score > 90% in warmup dashboard.

**Lemlist**: Settings > Email Providers > Add. Connect via SMTP/IMAP or OAuth. Configure sending signature. Enable lemwarm for ongoing warmup.

**Apollo**: Settings > Email > Connect Mailbox. OAuth for Google/Outlook. Set sending limits under Mailbox Settings. Enable auto-warmup if available.

**HubSpot**: Settings > General > Email. Connect personal inbox. Note: HubSpot Sequences send from personal inbox, not marketing email.

**Verify**: All accounts show "Connected" status. Warmup score > 80%. SPF/DKIM/DMARC all passing.
**If failed**: Check DNS records. Re-authenticate OAuth. Wait for warmup to complete if score < 80%.

### Step 2: Create Campaign and Configure Settings

**Duration**: 10 minutes
**Tool**: Campaign builder

Create a new campaign/sequence with these settings:

- **Name**: `{ICP segment}_{date}_{variant}` (e.g., `SaaS_VPs_20260311_v1`)
- **Sending schedule**: Monday-Friday, 8am-6pm in recipient's timezone
- **Daily sending limit**: Start at 30-50 per account, ramp 10-20% weekly
- **Stop on reply**: Enabled (remove from sequence when they reply)
- **Tracking**: Open tracking ON, link tracking OFF (link tracking hurts deliverability)

**Instantly-specific**: Set campaign schedule, enable "Optimize Send Time", add tags for filtering.
**Lemlist-specific**: Set sequence type (email only or multichannel), configure sending window.
**Apollo-specific**: Set sequence type, delivery window, and throttling rules.

**Verify**: Campaign created in draft mode. Settings match specifications above.
**If failed**: Double-check sending schedule timezone. Ensure daily limits don't exceed warmup capacity.

### Step 3: Build Sequence Steps with A/B Variants

**Duration**: 15-20 minutes
**Tool**: Sequence editor

Build a 3-5 step sequence with 2-day minimum spacing between steps:

**Recommended sequence structure:**
- Step 1 (Day 1): Initial outreach — 2-4 subject line variants, personalized body
- Step 2 (Day 3): Follow-up — different angle, reference Step 1
- Step 3 (Day 6): Value add — share relevant resource/case study
- Step 4 (Day 10): Breakup email — final attempt, clear CTA

For each step, create A/B variants:
- **Variant A**: Direct ask approach
- **Variant B**: Value-first approach
- Test ONE variable per step (subject line OR body, not both)
- Set even split (50/50 for 2 variants, 33/33/33 for 3)
- Winning metric: Reply rate (not open rate)

**Instantly A/B setup**: Add step > Variants > Add Variant B. Each variant gets its own subject + body.
**Lemlist A/B setup**: Steps > Add variant. Templates come with A/B versions. Set distribution %.
**Apollo A/B setup**: Edit step > Add Template > Set percentage allocation per variant.

**Verify**: Each step has 2+ variants. Personalization variables render correctly in preview. Delays between steps are 2+ days.
**If failed**: Preview with sample lead data. Fix broken variables. Ensure no step has 0-day delay.

### Step 4: Import Lead List

**Duration**: 5-10 minutes
**Tool**: Lead import section

Upload the formatted CSV from the lead list output formatting recipe.

**Instantly**: Campaign > Leads > Upload. Map columns to Instantly fields (Email required, First Name, Last Name, Company Name, custom Personalization). Deduplicate against global blocklist.

**Lemlist**: Campaign > Import leads > Upload CSV. Map columns to Lemlist variables. Enable deduplication against all campaigns.

**Apollo**: Sequence > Add contacts > Import CSV or select from Apollo database. Map fields. Check against suppression list.

**HubSpot**: Sequence > Add contacts > Select from CRM or import list. Maximum 500 contacts per sequence enrollment.

**Verify**: Lead count matches source file. No import errors. Personalization variables populated for all leads.
**If failed**: Check column header mapping. Fix encoding issues (re-save as UTF-8). Remove leads with missing required fields.

### Step 5: Activate Campaign and Monitor

**Duration**: 5 minutes to activate, ongoing monitoring
**Tool**: Campaign dashboard

Review campaign summary: lead count, sequence steps, A/B variants, sending schedule, daily limits. Activate the campaign.

**First 24-48 hours monitoring checklist:**
- Bounce rate should be < 3%. If > 5%, pause immediately and verify email list.
- Open rate should be 40-70% (with open tracking). Below 30% indicates deliverability issues.
- Reply rate baseline: 2-5% in first week is normal. Below 1% after 100+ sends suggests copy issues.
- Unsubscribe rate should be < 1%. Above 2% indicates targeting problems.

**Verify**: Campaign shows "Active" status. First emails sending within expected window. No immediate bounces.
**If failed**: Check sending account connection. Verify warmup is still active. Review spam folder placement with tools like mail-tester.com.

## Quality Benchmarks

| Quality Metric | Minimum Acceptable | Good | Excellent |
|---------------|-------------------|------|-----------|
| Open rate | > 40% | > 55% | > 70% |
| Reply rate | > 2% | > 5% | > 10% |
| Bounce rate | < 5% | < 3% | < 1% |
| Unsubscribe rate | < 2% | < 0.5% | < 0.1% |
| Meeting booking rate | > 0.5% | > 1.5% | > 3% |

**If below minimum**: Pause campaign. For low open rates, check deliverability (warmup score, SPF/DKIM). For low reply rates, revise copy and personalization. For high bounces, re-verify email list.

## Error Handling

| Error | Likely Cause | Recovery Action |
|-------|-------------|----------------|
| Emails landing in spam | Domain not warmed, missing DNS records | Pause campaign, verify SPF/DKIM/DMARC, complete warmup (14+ days) |
| 0% open rate | Tracking pixel blocked or deliverability crisis | Test with personal email, check mail-tester.com score, reduce daily volume |
| High bounce rate (>5%) | Unverified email list or expired addresses | Pause campaign, re-verify list, remove all bounced addresses |
| Sending account disconnected | OAuth token expired | Reconnect account in settings, re-authorize OAuth |
| Personalization variables showing raw text | Column mapping error during import | Re-import leads with correct column-to-variable mapping |
| Campaign not sending | Daily limit reached or scheduling conflict | Check sending schedule, increase daily limit gradually, verify timezone settings |

## Anti-Patterns

### Wrong: Launching cold outreach from primary business domain
Sending cold emails from `yourcompany.com` risks blacklisting your primary domain. If deliverability drops, all business email (customer support, invoices, internal) is affected. [src5]

### Correct: Use dedicated outreach domains
Register a separate domain (e.g., `try-yourcompany.com` or `meet-yourcompany.com`). Set up proper DNS records. Warm it for 14-21 days. If it gets burned, your primary domain is unaffected.

### Wrong: Testing multiple variables simultaneously in A/B tests
Changing both subject line and body text between variants makes it impossible to determine which change drove the result. [src6]

### Correct: Test one variable at a time
In Step 1, test subject lines only (identical body). In Step 2, test body copy (identical subject). Wait for 200+ sends per variant before declaring a winner. Use reply rate, not open rate, as the winning metric.
