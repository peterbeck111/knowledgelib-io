# Cloud Infrastructure Cost Benchmarks 2026

## Summary

Cloud infrastructure costs in 2026 show AWS, Azure, and GCP converging on similar pricing for most workloads, with differentiation driven by CPU architecture (Arm vs x86), discount programs, and data transfer policies. Organizations spend a median of $12M+ annually on public cloud, with 27% wasted on average. Arm-based instances deliver 20-50% cost savings over x86 equivalents across all three providers. [src1]

**Data vintage**: Based on Q4 2025 published pricing, Flexera State of Cloud report (750+ organizations), and cross-provider benchmarks.
**Key shift**: AWS doubled inter-AZ data transfer fees and raised cross-region costs 25-40% in 2025. Arm instances (Graviton, Cobalt, Axion) now the default cost-optimization choice. [src1, src4]

## Constraints
<!-- Agents: read before citing any benchmark number. -->

- Cloud pricing changes constantly. AWS averages 197 monthly price changes, Azure changes nearly monthly, GCP quarterly. Verify before committing.
- On-demand prices are rack rates. Most enterprises use reserved instances or savings plans at 30-72% discount. These benchmarks show on-demand unless noted.
- Region matters. Same instance type costs 10-30% more in EU or APAC vs US East regions.
- Data transfer (egress) costs are the hidden budget-breaker. Compute and storage get attention, but egress often surprises.
- 27% of cloud spend is wasted on average. Benchmarks assume reasonable optimization, not perfect efficiency.

## Compute Pricing

### General Purpose Instances (On-Demand)

**Definition**: Hourly cost for general-purpose compute instances with balanced CPU, memory, and networking. Compared on equivalent configurations (4 vCPU, 16GB RAM) in US East regions.

| Provider | Instance Family | Architecture | $/hour (4 vCPU, 16GB) | Monthly (730 hrs) |
|----------|----------------|-------------|----------------------|-------------------|
| AWS | m7g (Graviton3) | Arm | $0.1632 | $119.14 |
| AWS | m7i (Intel) | x86 | $0.2016 | $147.17 |
| Azure | D4as v5 (Ampere) | Arm | $0.1540 | $112.42 |
| Azure | D4s v5 (Intel) | x86 | $0.1920 | $140.16 |
| GCP | t2a-standard-4 (Axion) | Arm | $0.1508 | $110.08 |
| GCP | n2-standard-4 (Intel) | x86 | $0.1942 | $141.77 |

**Trend**: Arm instances 15-25% cheaper than x86 equivalents on all providers. Azure has the largest gap (65% on-demand, 69% spot). GCP often has lowest on-demand prices; AWS Graviton dominates compute-optimized. [src1, src5]
**Red flag threshold**: Running x86 general-purpose instances without evaluating Arm alternatives is leaving 15-25% savings on the table.

### Discount Programs

**Definition**: Committed-use and savings plan discounts compared across providers.

| Discount Model | AWS | Azure | GCP |
|----------------|-----|-------|-----|
| 1-Year Reserved/Committed | 30-40% off | 30-38% off | 25-37% off |
| 3-Year Reserved/Committed | 55-72% off | 50-65% off | 50-70% off |
| Spot/Preemptible | Up to 90% off | Up to 90% off | Up to 80% off |
| Sustained Use (auto) | N/A | N/A | Up to 30% off |
| Savings Plans (flexible) | 20-35% off | 20-30% off | N/A |

**Trend**: AWS and Azure offer up to 72% off with 3-year commitments. GCP's sustained use discounts apply automatically. Spot pricing volatile but offers massive savings for fault-tolerant workloads. [src1, src6]
**Red flag threshold**: Running production workloads entirely on on-demand pricing signals a FinOps maturity problem.

## Storage Pricing

### Object Storage (Hot/Standard Tier)

**Definition**: Per-GB-per-month cost for standard object storage (S3, Blob, GCS) in US regions. Includes storage only; excludes API calls and egress.

| Provider | Service | $/GB/Month | 100 TB/Month |
|----------|---------|-----------|--------------|
| AWS | S3 Standard | $0.023 | $2,300 |
| Azure | Blob Hot | $0.0184 | $1,840 |
| GCP | Cloud Storage Standard | $0.020 | $2,000 |

**Trend**: Azure Blob hot storage 20% cheaper than AWS S3 at the base tier. GCP competitive in the middle. [src2]
**Red flag threshold**: Storing > 50% of data in hot storage without lifecycle policies typically wastes 30-50% of storage budget.

## Common Misinterpretations

- **Comparing on-demand prices as if anyone pays them**: Enterprise customers on 3-year reservations pay 50-72% less than on-demand rates. On-demand comparisons are starting points, not actual costs. [src1]
- **Ignoring data transfer in cost comparisons**: A compute-cheaper provider becomes expensive if your architecture generates heavy cross-region or egress traffic. Always model the full stack.
- **Treating all regions as equivalent**: US East is consistently cheapest. EU and APAC regions add 10-30% to the same instance types. Factor region into any comparison.
- **Assuming multi-cloud saves money**: Multi-cloud adds 10-20% management overhead. It's a risk/vendor-lock strategy, not a cost optimization strategy.
