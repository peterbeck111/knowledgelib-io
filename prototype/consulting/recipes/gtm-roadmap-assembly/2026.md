# GTM Roadmap Assembly

## Purpose

This recipe executes Module 8 of the Rorschach GTM curriculum: assembling all prior module outputs — signal library, friction gates, committee maps, and counterfactual templates — into a unified go-to-market plan with 90-day milestones, named owner accountability, leading-indicator success metrics, and a scoped advisory retainer at $3K/month. This is the capstone deliverable that transforms individual GTM components into an executable system. [src1, src2]

## Prerequisites
<!-- Agents: verify ALL prerequisites before executing. Missing prerequisites = failed execution. -->

- [ ] **Signal Library complete** — documented exhaust fume signals with detection triggers and response protocols (Module 2)
- [ ] **Friction Gate Blueprint complete** — designed and prototyped friction gates with filter rate projections (Module 4)
- [ ] **Committee Maps complete** — buying committee structures mapped for target accounts with influence pathways (Module 5)
- [ ] **Counterfactual Scenario Package complete** — personalized failure simulations, mock incident reports, and entropy trajectories for 3 target accounts (Module 6)
- [ ] **CRM access** — pipeline tracking capability, even if just a spreadsheet

## Constraints
<!-- Hard rules. Agents: enforce throughout execution. Violating these = broken output or legal risk. -->

- All four component libraries must be complete before assembly. Integrating partial components produces a plan that looks complete but has structural holes — the consulting equivalent of Swiss Cheese. [src1]
- 90-day milestones must be specific, measurable, and have a single named owner. "Improve pipeline" is not a milestone. "Generate 12 qualified opportunities via friction gate deployment by Day 60" is. [src3]
- Success metrics must be leading indicators measurable within 90 days. Revenue is a lagging indicator — use pipeline velocity, qualification rate, friction gate completion rate, and counterfactual engagement rate instead. [src2]
- Advisory retainer is scoped to deliverables, not hours. Hourly billing creates perverse incentives (client minimizes hours consumed rather than maximizing value extracted). Deliverable-based retainers align consultant and client incentives. [src4]
- The retainer price of $3K/month is calibrated for solo consultants targeting mid-market clients. Adjust upward for enterprise ($5K-$10K/month) or downward for SMB ($1.5K-$2K/month).

## Execution Flow

### Step 1: Component Integration Audit

**Duration**: 60 minutes
**Tool**: Checklist + document review

Audit all four component libraries for completeness and internal consistency:
- **Signal Library**: Are detection triggers specific enough to automate or semi-automate? Does each signal have a clear response protocol?
- **Friction Gates**: Are filter rate projections backed by data? Does each gate deliver standalone value?
- **Committee Maps**: Are influence pathways documented for each target account? Are hidden decision-makers identified?
- **Counterfactual Templates**: Are scenarios account-specific (not generic)? Are mock incident reports labeled as simulations?

Cross-check for integration gaps: Does each signal have a corresponding friction gate? Does each committee map have a corresponding counterfactual? Do friction gates align with committee structures (e.g., multi-stakeholder gate matches multi-person buying committee)?

**Verify**: All four components pass audit with no critical gaps. Integration points mapped.
**If failed**: Identify missing components and complete them before proceeding. Partial assembly produces unusable plans.

### Step 2: Build 90-Day Milestone Framework

**Duration**: 90 minutes
**Tool**: Project management tool or structured document

Define 90-day milestones using the OKR framework [src3] adapted for GTM execution:

**Days 1-30: Foundation**
- Deploy friction gates (diagnostic tool live, operational calculator live, multi-stakeholder workshop template ready)
- Activate signal detection for top 10 exhaust fume signals
- Complete committee mapping for 3 priority target accounts
- Milestone owner: [Name]
- Key result: Friction gates operational, first 5 prospects entering gate funnel

**Days 31-60: Activation**
- Deliver personalized counterfactual scenarios to 3-5 qualified prospects (post-friction-gate)
- Run first multi-stakeholder workshops with 2+ target accounts
- Calibrate friction gate filter rates against actual data (target: 80% filter, 5x conversion)
- Milestone owner: [Name]
- Key result: 8-12 qualified opportunities in pipeline, 2+ proposals delivered

**Days 61-90: Conversion + Retainer**
- Close 2-3 initial engagements from qualified pipeline
- Transition 1-2 completed engagements to advisory retainer
- Measure and report: actual filter rate, qualified conversion rate, pipeline velocity, retainer conversion rate
- Milestone owner: [Name]
- Key result: $25K-$75K revenue closed, 1+ retainer signed at $3K/month

**Verify**: Every milestone has a specific key result, a named owner, and a measurable metric. No milestone spans more than 30 days.
**If failed**: If milestones are too vague, apply the "would a stranger know if this was achieved?" test. If not, make it more specific.

### Step 3: Assign Owner Accountability

**Duration**: 30 minutes
**Tool**: RACI matrix (spreadsheet)

For each milestone and sub-task, assign a single owner using RACI:
- **R (Responsible)**: Single person who does the work — one person only, never shared
- **A (Accountable)**: Person who approves/reviews — typically the engagement lead
- **C (Consulted)**: People whose input is needed before action
- **I (Informed)**: People who need to know after action is taken

For solo consultants: R and A are the same person. C and I may include clients, partners, or virtual assistants.

**Verify**: Every task has exactly one R. No task has multiple R assignments. [src3]
**If failed**: If a task "needs" two owners, split it into two tasks.

### Step 4: Define Success Metrics Dashboard

**Duration**: 45 minutes
**Tool**: Spreadsheet or dashboard tool

Build a leading-indicator dashboard with weekly tracking:

| Metric | Week 1-4 Target | Week 5-8 Target | Week 9-12 Target | Source |
|--------|----------------|-----------------|-------------------|--------|
| Signal detections (exhaust fumes identified) | 10-15 | 15-25 | 20-30 | Signal library alerts |
| Friction gate entries (prospects entering) | 5-8 | 8-15 | 10-20 | Form/tool analytics |
| Friction gate completions (prospects passing) | 1-2 | 2-4 | 3-5 | Form/tool analytics |
| Filter rate (% filtered out) | Calibrating | 70-85% | 75-85% | Calculated |
| Qualified opportunities generated | 2-4 | 4-8 | 6-12 | CRM |
| Counterfactual deliveries | 0-1 | 2-3 | 3-5 | Outreach tracking |
| Proposals delivered | 0-1 | 1-3 | 2-4 | CRM |
| Retainer conversations initiated | 0 | 1-2 | 2-3 | CRM |

**Verify**: Every metric is measurable from existing tools (no new infrastructure required). Dashboard is updated weekly. [src2]
**If failed**: If a metric cannot be measured with current tools, either add a simple tracking mechanism or replace with a measurable proxy.

### Step 5: Scope Advisory Retainer

**Duration**: 60 minutes
**Tool**: Proposal template

Design the advisory retainer offering at $3K/month (adjust for market segment) with:

**Retainer deliverables (monthly)**:
- Signal monitoring report: ongoing exhaust fume detection with recommended responses
- Friction gate performance analysis: filter rates, conversion metrics, optimization recommendations
- Committee map updates: changes in buying committee structures for active accounts
- 2 hours of strategic advisory calls (scheduled, not ad-hoc)
- Quarterly counterfactual refresh: updated entropy trajectories for active accounts

**Retainer SLA**:
- Response time: 24 hours for questions, 48 hours for analysis requests
- Monthly report delivered by 5th business day of following month
- Quarterly business review (60 minutes) included

**Retainer pricing rationale** [src4]:
- $3K/month = $36K/year — positioned as <10% of the value delivered (minimum value: one additional closed deal per quarter = $50K+ annual impact)
- Deliverable-based, not hourly — client gets the outputs regardless of hours consumed
- 3-month minimum commitment with monthly renewal thereafter
- Onboarding month includes initial signal library setup + friction gate deployment (may require additional one-time fee of $5K-$10K)

**Verify**: Retainer proposal passes the "would I buy this?" test. Value delivered exceeds price by minimum 3x.
**If failed**: If value proposition is unclear, add a concrete ROI calculation using the client's actual pipeline metrics.

### Step 6: Assemble Final GTM Document

**Duration**: 60 minutes
**Tool**: Document tool (Google Docs, Notion, or Markdown)

Compile the unified GTM roadmap document integrating all outputs:

1. **Executive Summary**: One-page overview of the 90-day GTM strategy, target outcomes, and investment required
2. **Component Library References**: Links/references to signal library, friction gate blueprints, committee maps, counterfactual packages
3. **90-Day Milestone Plan**: Full milestone framework with dates, owners, and key results
4. **Success Metrics Dashboard**: Embedded or linked weekly tracking dashboard
5. **Advisory Retainer Proposal**: Standalone section or separate document for client-facing use
6. **Risk Register**: Top 5 risks to the 90-day plan with mitigation strategies

**Verify**: Document is complete, internally consistent, and actionable. A new team member could execute from this document alone.
**If failed**: If the document requires verbal explanation to be understood, rewrite for clarity. The document should stand alone. [src5]

## Error Handling

| Error | Likely Cause | Recovery Action |
|-------|-------------|----------------|
| Components incomplete — missing one or more libraries | Workshop modules skipped or partially completed | Complete missing module before assembly — do not assemble from partial components |
| Milestones are vague ("improve pipeline") | Lack of current baseline metrics | Establish baseline metrics first: current inquiry volume, qualification rate, close rate, deal size |
| No single owner for milestones | Team avoids accountability assignment | Force single-owner constraint: split shared tasks into individual tasks |
| Retainer price objection anticipated | Value proposition not concrete enough | Add ROI calculation using prospect's actual pipeline metrics and close rates |
| 90-day timeline unrealistic | Signal library or friction gates require more build time | Extend to 120-day plan or reduce scope of first phase to "foundation only" |

## Anti-Patterns

### Wrong: Assembling the roadmap before all components are complete
Rushing to produce the GTM document with placeholder sections for incomplete components. Result: plan has structural holes that only become apparent during execution, requiring costly mid-stream redesign. [src1]

### Correct: Complete all four component libraries before assembly
The integration step requires all components because cross-references between them (signal triggers friction gate, friction gate routes to committee map, committee map selects counterfactual) cannot be validated with missing pieces.

### Wrong: Setting milestones as activities rather than outcomes
"Run 10 outreach campaigns" is an activity. "Generate 12 qualified opportunities" is an outcome. Activity-based milestones reward busywork, not results. [src3]

### Correct: Define every milestone as a measurable outcome
Use the OKR framework: Objective (qualitative direction) + Key Result (quantitative measurement). If the key result cannot be measured with existing tools, either add tracking or choose a different metric.

### Wrong: Pricing the advisory retainer by the hour
$3K/month divided by expected hours creates an implicit hourly rate that clients will benchmark against freelancer rates. Result: price pressure, scope creep ("I'm paying for 10 hours, I want 10 hours"), and misaligned incentives. [src4]

### Correct: Price the retainer by deliverables
$3K/month buys: monthly signal report, friction gate analysis, committee map updates, 2 advisory calls, quarterly counterfactual refresh. The client pays for outputs, not inputs. If the consultant becomes more efficient, they keep the margin — this rewards expertise.
