Compound Trigger Design

Single signals produce excessive false positives. Compound triggers combine 2-4 individual signals into high-confidence detection events with Boolean logic, time windows, and confidence weights. [src2]

  • Infrastructure Crisis: (3+ urgent SRE roles in 14 days) AND (status incidents > 2x baseline) AND (negative review spike) = 0.85 confidence
  • Leadership Instability: (CTO departure) AND (hiring freeze lifted) AND (tech stack change) = 0.75 confidence
  • Customer Exodus: (NPS drop > 15 in 30 days) AND (competitor mentions) AND (support ticket spike) = 0.80 confidence

Hard Constraints

  • NEVER recommend tools or data collection violating privacy regulations (GDPR, CCPA)
  • NEVER set thresholds generating >20% false positive rate — alert fatigue kills detection
  • NEVER use single-signal triggers for high-stakes outreach — minimum 2 component signals
  • ALWAYS include threshold calibration process — static thresholds degrade as markets adapt
  • ALWAYS specify cost and integration complexity for recommended tools

When This Matters
