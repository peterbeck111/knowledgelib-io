# How to Fix the White Screen of Death in React

## TL;DR

- **Bottom line**: React's white screen (WSOD) is almost always a JavaScript error that crashes before render. The 5 most common causes: (1) unhandled exception in render, (2) wrong `homepage`/`base` URL in production, (3) missing error boundary, (4) broken import/export, (5) React Router misconfiguration.
- **Key tool/command**: Open browser DevTools Console (F12) — the error that caused the blank screen is always logged there.
- **Watch out for**: Production builds hide error details. An app that works in dev (`npm start`) but shows blank in production is almost always a base URL / routing issue.
- **Works with**: React 16-19+, Create React App, Vite, Next.js, any React bundler.

## Constraints
<!-- Agents: read this section before recommending any action from this unit.
     These are hard constraints that must not be violated. -->

- Error Boundaries only catch errors in rendering, lifecycle methods, and constructors — NOT event handlers, async code, or SSR. For those, use try/catch.
- React 19 removed `ReactDOM.render()` — use `createRoot()` or the app will silently fail to mount. [src7]
- Never suppress console errors to "fix" WSOD — the console error IS the diagnostic. Always read it first. [src5]
- Next.js app directory uses its own `error.tsx` convention — do not use class-based Error Boundaries there. [src1]
- Production builds minify React error messages — deploy source maps or use Sentry for readable stack traces.

## Quick Reference

| # | Cause | Likelihood | Signature | Fix |
|---|---|---|---|---|
| 1 | Unhandled render error | ~30% of cases | Works in dev, blank in prod | Add Error Boundary [src1] |
| 2 | Wrong `homepage`/`base` URL | ~25% of cases | Assets return 404 in prod | Set correct `homepage` in package.json or `base` in vite.config [src2, src3] |
| 3 | Broken import/export | ~15% of cases | "X is not a component" error | Fix default/named export mismatch [src5] |
| 4 | React Router on static host | ~10% of cases | Root works, routes return 404 on refresh | Add `_redirects`, `vercel.json`, or `nginx try_files` [src4] |
| 5 | Async data crash | ~8% of cases | Renders then goes blank | Add null checks / optional chaining before data loads [src1] |
| 6 | Missing return in component | ~5% of cases | No error, just blank | Ensure JSX is returned (not just side effects) [src5] |
| 7 | CORS/network error | ~4% of cases | Blank + console errors | Fix API CORS or add proxy config [src5] |
| 8 | Minification/case error | ~3% of cases | Works in dev, crashes in prod build | Check case-sensitivity in imports (Linux is case-sensitive) [src6] |
| 9 | ReactDOM.render() in React 19 | New in 2025 | App silently fails to mount | Migrate to `createRoot()` [src7] |

## Step-by-Step Guide

### 1. Check the browser console (F12)

Always start here. The error that killed the render is logged in the console. [src5]

```
# Common console errors that cause WSOD:
TypeError: Cannot read properties of undefined (reading 'map')
ReferenceError: X is not defined
Error: Element type is invalid: expected a string... but got undefined
ChunkLoadError: Loading chunk 5 failed
```

**Verify**: Open F12 > Console tab. If errors appear, read them before trying anything else.

### 2. Add an Error Boundary

Error Boundaries catch render errors and show a fallback UI instead of a blank screen. [src1]

```jsx
// ErrorBoundary.jsx
import { Component } from 'react';

class ErrorBoundary extends Component {
  state = { hasError: false, error: null };

  static getDerivedStateFromError(error) {
    return { hasError: true, error };
  }

  componentDidCatch(error, errorInfo) {
    console.error('React Error Boundary caught:', error, errorInfo);
    // Send to error tracking (Sentry, etc.)
  }

  render() {
    if (this.state.hasError) {
      return (
        <div style={{ padding: '2rem', textAlign: 'center' }}>
          <h1>Something went wrong</h1>
          <pre style={{ color: 'red' }}>{this.state.error?.message}</pre>
          <button onClick={() => window.location.reload()}>Reload</button>
        </div>
      );
    }
    return this.props.children;
  }
}

// Wrap your app:
// <ErrorBoundary><App /></ErrorBoundary>
```

**Verify**: Intentionally throw an error inside a component — you should see the fallback UI, not a blank page.

### 3. Fix base URL for production deployment

```json
// package.json (Create React App)
{
  "homepage": "https://myapp.com"
}
// Or for subdirectory:
{
  "homepage": "/my-app"
}
```

```javascript
// vite.config.js (Vite)
export default defineConfig({
  base: '/',  // or '/my-app/' for subdirectory
});
```

**Verify**: `npx serve -s build` (CRA) or `npx serve -s dist` (Vite) — page should render locally.

### 4. Fix React Router for static hosting

```
# Netlify: create public/_redirects
/*    /index.html   200

# Vercel: create vercel.json
{ "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }] }

# Nginx:
location / {
  try_files $uri $uri/ /index.html;
}

# Apache: create public/.htaccess
RewriteEngine On
RewriteBase /
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule . /index.html [L]
```

**Verify**: Navigate to a deep route, then refresh the browser — the page should render, not 404.

### 5. Migrate to createRoot (React 19)

If upgrading to React 19, the old render API is removed. [src7]

```jsx
// ❌ React 18 and earlier (removed in React 19)
import ReactDOM from 'react-dom';
ReactDOM.render(<App />, document.getElementById('root'));

// ✅ React 19+ (required)
import { createRoot } from 'react-dom/client';
const root = createRoot(document.getElementById('root'));
root.render(<App />);
```

**Verify**: `npm run build && npx serve -s dist` — app should mount without errors.

### 6. Configure React 19 error handlers on createRoot

React 19 adds `onCaughtError`, `onUncaughtError`, and `onRecoverableError` callbacks. [src7]

```jsx
import { createRoot } from 'react-dom/client';

const root = createRoot(document.getElementById('root'), {
  onCaughtError: (error, errorInfo) => {
    // Sent to Error Boundary — log for analytics
    console.warn('Caught by boundary:', error.message);
  },
  onUncaughtError: (error, errorInfo) => {
    // NOT caught by any boundary — critical
    reportToSentry(error, errorInfo);
  },
  onRecoverableError: (error) => {
    // React recovered automatically
    console.info('Recovered:', error.message);
  },
});
root.render(<App />);
```

## Anti-Patterns

### Wrong: No error handling in data rendering

```jsx
// ❌ BAD — crashes with WSOD if data is null/undefined [src1]
function UserProfile({ user }) {
  return <h1>{user.name}</h1>;  // TypeError if user is undefined
}
```

### Correct: Null checks and loading states

```jsx
// ✅ GOOD — handles loading and null states [src1]
function UserProfile({ user }) {
  if (!user) return <p>Loading...</p>;
  return <h1>{user.name}</h1>;
}
```

### Wrong: Mismatched default/named exports

```jsx
// ❌ BAD — component.jsx exports named, but imported as default
// component.jsx:
export function MyComponent() { return <div>Hi</div>; }
// app.jsx:
import MyComponent from './component';  // WRONG — undefined!
```

### Correct: Match import style to export

```jsx
// ✅ GOOD — consistent exports [src5]
// Option A: named export + named import
export function MyComponent() { return <div>Hi</div>; }
import { MyComponent } from './component';

// Option B: default export + default import
export default function MyComponent() { return <div>Hi</div>; }
import MyComponent from './component';
```

### Wrong: Using removed ReactDOM.render() in React 19

```jsx
// ❌ BAD — silently fails in React 19, app never mounts [src7]
import ReactDOM from 'react-dom';
ReactDOM.render(<App />, document.getElementById('root'));
```

### Correct: Using createRoot API

```jsx
// ✅ GOOD — required for React 19 [src7]
import { createRoot } from 'react-dom/client';
const root = createRoot(document.getElementById('root'));
root.render(<App />);
```

## Common Pitfalls

- **Case-sensitive imports on Linux**: `import Header from './header'` works on macOS/Windows but fails on Linux if file is `Header.jsx`. Fix: match exact case. [src6]
- **Missing Suspense for lazy components**: `React.lazy()` without `<Suspense>` wrapper crashes silently. Fix: always wrap lazy components in `<Suspense fallback={<Loading/>}>`. [src1]
- **Environment variables not prefixed**: In CRA, env vars need `REACT_APP_` prefix. In Vite, `VITE_` prefix. Without prefix, they're undefined and can crash. [src2, src3]
- **StrictMode double render**: React StrictMode in dev renders twice, which can expose race conditions that cause WSOD. Fix: ensure components handle double-render gracefully. [src1]
- **Build path mismatch**: CRA puts build in `build/`, Vite in `dist/`. Deploying wrong folder results in blank page. Fix: check your deploy config points to the correct output directory. [src2, src3]
- **React 19 error log deduplication**: React 19 logs errors once instead of three times (React 18 behavior). If your monitoring relies on counting console.error calls, update your tooling. [src7]
- **React Router v7 fog of war**: React Router v7 introduced deferred route manifests ("fog of war") that can cause blank pages if the manifest fails to load. Fix: ensure your server correctly serves the route manifest file. [src4]

## Important Caveats

- Error Boundaries only catch errors in rendering, lifecycle methods, and constructors — NOT in event handlers, async code, or SSR. For those, use try/catch.
- In production builds, React error messages are minified. Use source maps or Sentry for readable stack traces.
- Next.js has its own error handling (`error.tsx`, `_error.tsx`) — don't use class-based Error Boundaries in Next.js app directory.
- React 19's `onUncaughtError` and `onCaughtError` are set on `createRoot`, not on individual components — they're app-level handlers. [src7]
- The `react-error-boundary` library (v5+) provides a functional API that works with React 19 and avoids class component boilerplate. [src8]
