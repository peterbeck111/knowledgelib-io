# Dashboard Template Library

## Purpose

This recipe produces a library of pre-built dashboard templates organized by startup business model (SaaS, marketplace, ecommerce, services), each with default KPIs, widget layouts, SQL calculation queries, and embedded industry benchmarks. The output enables any startup to deploy a functional metrics dashboard in 1-3 hours by selecting their business model template, connecting data sources, and customizing from a working baseline rather than starting from scratch. [src1]

## Prerequisites
<!-- Agents: verify ALL prerequisites before executing. Missing prerequisites = failed execution. -->

- [ ] **Business model identified** — SaaS/subscription, marketplace/platform, ecommerce/DTC, or services/agency
- [ ] **PostgreSQL database** with at least 3 months of transactional data — [Supabase](https://supabase.com) or [Neon](https://neon.tech)
- [ ] **Dashboard tool** — [Retool](https://retool.com), [Metabase](https://www.metabase.com), or custom React framework
- [ ] **Primary data source connected** — Stripe (SaaS), Shopify (ecommerce), or custom payment system (marketplace)
- [ ] **Revenue model parameters** — pricing structure, billing frequency, commission/take rates

## Constraints
<!-- Hard rules. Agents: enforce throughout execution. -->

- Templates are starting points. Customize KPIs within 30 days based on actual data availability and team priorities. Stale default templates mislead decision-making.
- KPI formulas assume standard definitions (e.g., LTV = ARPU / monthly churn rate). Verify calculations match your accounting methodology before presenting to investors. [src1]
- Benchmarks are industry medians from 2025-2026 data. Show top-quartile and bottom-quartile ranges alongside to avoid false confidence. [src6]
- Widget layouts designed for 1280px+ desktop. Mobile layouts require separate breakpoint configuration in the dashboard tool.
- Data source connections are template-specific. Swapping data sources (e.g., Stripe to Paddle) requires query modifications — templates are not data-source-agnostic.

### Step 2: Configure KPI Calculations

**Duration**: 30-60 minutes
**Tool**: SQL

Template A — SaaS KPI queries:

```sql
-- MRR (Monthly Recurring Revenue)
SELECT
  DATE_TRUNC('month', period_start) as month,
  SUM(amount / EXTRACT(MONTH FROM AGE(period_end, period_start))) as mrr
FROM subscriptions
WHERE status = 'active'
GROUP BY month ORDER BY month;

-- Net Revenue Retention (NRR)
WITH cohort_revenue AS (
  SELECT customer_id,
    DATE_TRUNC('month', first_payment) as cohort_month,
    DATE_TRUNC('month', payment_date) as payment_month,
    SUM(amount) as revenue
  FROM payments
  GROUP BY customer_id, cohort_month, payment_month
)
SELECT cohort_month,
  SUM(revenue) FILTER (WHERE payment_month = cohort_month + INTERVAL '12 months')
  / NULLIF(SUM(revenue) FILTER (WHERE payment_month = cohort_month), 0) * 100 as nrr_12mo
FROM cohort_revenue GROUP BY cohort_month;

-- Logo Churn Rate (monthly)
SELECT month,
  churned_customers::NUMERIC / NULLIF(start_customers, 0) * 100 as churn_rate
FROM (
  SELECT DATE_TRUNC('month', churned_at) as month,
    COUNT(*) as churned_customers,
    (SELECT COUNT(*) FROM customers WHERE created_at < DATE_TRUNC('month', c.churned_at)
     AND (churned_at IS NULL OR churned_at >= DATE_TRUNC('month', c.churned_at))) as start_customers
  FROM customers c WHERE churned_at IS NOT NULL
  GROUP BY month
) sub;
```

Template B — Marketplace KPI queries:

```sql
-- GMV (Gross Merchandise Value)
SELECT DATE_TRUNC('month', completed_at) as month,
  SUM(transaction_amount) as gmv,
  SUM(platform_fee) as net_revenue,
  ROUND(SUM(platform_fee) / NULLIF(SUM(transaction_amount), 0) * 100, 2) as take_rate_pct
FROM transactions
WHERE status = 'completed'
GROUP BY month ORDER BY month;

-- Liquidity (% of listings that result in a transaction within 30 days)
SELECT DATE_TRUNC('month', created_at) as month,
  COUNT(*) FILTER (WHERE first_transaction_at IS NOT NULL
    AND first_transaction_at - created_at <= INTERVAL '30 days')::NUMERIC
  / NULLIF(COUNT(*), 0) * 100 as liquidity_pct
FROM listings GROUP BY month;

-- Buyer:Seller Ratio
SELECT DATE_TRUNC('month', last_active) as month,
  COUNT(*) FILTER (WHERE user_type = 'buyer') as buyers,
  COUNT(*) FILTER (WHERE user_type = 'seller') as sellers,
  ROUND(COUNT(*) FILTER (WHERE user_type = 'buyer')::NUMERIC
    / NULLIF(COUNT(*) FILTER (WHERE user_type = 'seller'), 0), 2) as buyer_seller_ratio
FROM users WHERE last_active >= CURRENT_DATE - INTERVAL '12 months'
GROUP BY month;
```

Template C — Ecommerce KPI queries:

```sql
-- AOV (Average Order Value) and Conversion Rate
SELECT DATE_TRUNC('month', order_date) as month,
  COUNT(*) as orders,
  SUM(total_amount) as revenue,
  ROUND(AVG(total_amount), 2) as aov,
  ROUND(COUNT(*)::NUMERIC / NULLIF(sessions.count, 0) * 100, 2) as conversion_rate
FROM orders
LEFT JOIN (SELECT DATE_TRUNC('month', session_date) as month, COUNT(*) as count
  FROM sessions GROUP BY month) sessions USING (month)
GROUP BY month, sessions.count ORDER BY month;

-- ROAS by Channel
SELECT channel,
  SUM(revenue_attributed) as revenue,
  SUM(ad_spend) as spend,
  ROUND(SUM(revenue_attributed) / NULLIF(SUM(ad_spend), 0), 2) as roas
FROM marketing_attribution
WHERE attribution_date >= CURRENT_DATE - INTERVAL '30 days'
GROUP BY channel ORDER BY roas DESC;

-- Repeat Purchase Rate
SELECT DATE_TRUNC('month', first_order_date) as cohort_month,
  COUNT(*) as total_customers,
  COUNT(*) FILTER (WHERE order_count > 1) as repeat_customers,
  ROUND(COUNT(*) FILTER (WHERE order_count > 1)::NUMERIC / COUNT(*) * 100, 1) as repeat_rate
FROM (
  SELECT customer_id, MIN(order_date) as first_order_date, COUNT(*) as order_count
  FROM orders GROUP BY customer_id
) sub GROUP BY cohort_month;
```

**Verify**: All KPI queries return data for at least 3 months. No NULL values in primary metrics.
**If failed**: Check table names — templates assume standard naming. Map to your actual schema.

## Error Handling

| Error | Likely Cause | Recovery Action |
|-------|-------------|----------------|
| KPI shows $0 or NULL | Table name mismatch between template and actual schema | Map template table names to your schema; check column names |
| Benchmark bands not visible | Dashboard tool doesn't support conditional formatting | Add benchmark values as reference lines instead of background bands |
| Cohort heatmap empty | Insufficient historical data (< 6 months) | Switch to simple line chart until enough cohort data accumulates |
| Conversion funnel shows 100% at every stage | Funnel stages not ordered correctly in query | Verify stage ordering matches your actual user journey |
| ROAS shows negative values | Revenue attribution window too narrow | Extend attribution window from 7 to 30 days; check UTM parameter capture |
| Template import fails | Dashboard tool version incompatibility | Export as individual queries + manual widget creation as fallback |

## Anti-Patterns

### Wrong: Using a SaaS template for a marketplace business
Marketplace fundamentals (GMV, take rate, liquidity, match rate) are structurally different from SaaS metrics (MRR, churn, NRR). Forcing marketplace data into SaaS templates produces misleading KPIs — a marketplace with $1M GMV and 10% take rate shows $100K in "revenue," but SaaS benchmarks treat that as low-growth. [src5]

### Correct: Match template to business model, then customize
Select the template that matches your primary revenue model. Hybrid businesses (e.g., SaaS with marketplace component) should start with the dominant model and add KPIs from the secondary model as custom widgets.

### Wrong: Treating template benchmarks as targets
Industry median benchmarks describe average performance, not your target. A pre-seed startup comparing against Series B medians will appear to underperform. Benchmarks without stage context mislead founders and investors. [src6]

### Correct: Layer stage-appropriate benchmarks with your own targets
Display three reference points on each KPI: (1) your own target, (2) stage-matched median, (3) top-quartile for your stage. Set targets based on your fundraising plan, not on general industry data.
