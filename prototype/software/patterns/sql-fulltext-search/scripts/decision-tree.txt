├── Using PostgreSQL?
│   ├── YES
│   │   ├── Need typo/fuzzy tolerance?
│   │   │   ├── YES → Use pg_trgm with GIN(gin_trgm_ops) index (see Code Example 3)
│   │   │   └── NO ↓
│   │   ├── Need relevance ranking with field weighting?
│   │   │   ├── YES → Use tsvector + setweight() + ts_rank() (see Code Example 1)
│   │   │   └── NO ↓
│   │   ├── Simple keyword/phrase search?
│   │   │   ├── YES → Use to_tsvector() + to_tsquery() with GIN index (see Step 1-3)
│   │   │   └── NO ↓
│   │   └── Need both fuzzy AND ranked search?
│   │       └── YES → Combine tsvector + pg_trgm with fallback (see Code Example 4)
│   └── NO ↓
├── Using MySQL?
│   ├── YES
│   │   ├── Need precise inclusion/exclusion (+/- operators)?
│   │   │   ├── YES → Use BOOLEAN MODE (see Code Example 5)
│   │   │   └── NO ↓
│   │   ├── Simple relevance-ranked search?
│   │   │   ├── YES → Use NATURAL LANGUAGE MODE (see Step 4)
│   │   │   └── NO ↓
│   │   └── Need to find related terms from vague input?
│   │       └── YES → Use WITH QUERY EXPANSION (see Quick Reference row 9)
│   └── NO ↓
└── Dataset >10M docs OR need sub-50ms p99 OR complex faceting?
    └── YES → Use dedicated search engine (Elasticsearch, Typesense, Meilisearch)
